# Epic Story Packets

Create epic folders here only when real work grouping exists.

Suggested naming:

```text
E01-domain-name/
E02-domain-name/
E03-domain-name/
```

Create real epic names from accepted intake or product truth, not from generic placeholders.

## When Epic Folder Exists

Create an epic folder when:

- multiple related stories need one durable home
- product-doc links for a domain area should stay grouped
- high-risk packet or multiple normal stories will accumulate around one area

Do not create epic folders for hypothetical future domains.

## Minimum Expectation

An epic folder should contain either:

- one or more story files
- one or more high-risk story folders
- both, if the area needs mixed packet types

The folder itself does not need extra docs unless the work truly needs them.

## Relationship To Backlog

- backlog row names the epic and next slicing action
- epic folder contains the actual story packets once slicing begins
- if epic never becomes real work, keep it in backlog and do not create folder yet
